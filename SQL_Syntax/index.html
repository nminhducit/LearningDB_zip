SQL Server Basic Syntax

1. SELECT

The SELECT statement is used to select data from a database.

SELECT column1, column2, ...
FROM table_name;

Example:

SELECT * FROM employees;

2. INSERT

The INSERT statement is used to insert new records into a table.

INSERT INTO table_name (column1, column2, ...)
VALUES (value1, value2, ...);

Example:

INSERT INTO employees (name, age, department)
VALUES ('John Doe', 30, 'IT');

3. UPDATE

The UPDATE statement is used to modify the existing records in a table.

UPDATE table_name
SET column1 = value1, column2 = value2, ...
WHERE condition;

Example:

UPDATE employees
SET department = 'Finance'
WHERE name = 'John Doe';

4. DELETE

The DELETE statement is used to delete existing records in a table.

DELETE FROM table_name
WHERE condition;

Example:

DELETE FROM employees
WHERE age > 50;

5. CREATE TABLE

The CREATE TABLE statement is used to create a new table in the database.

CREATE TABLE table_name (
    column1 datatype constraint,
    column2 datatype constraint,
    ...
);

Example:

CREATE TABLE employees (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    age INT,
    department VARCHAR(50)
);

6. JOIN TABLE

The JOIN clause is used to combine rows from two or more tables based on a related column between them.

SELECT column1, column2, ...
FROM table1
JOIN table2
ON table1.column = table2.column;

Example:

SELECT employees.name, departments.department_name
FROM employees
JOIN departments
ON employees.department_id = departments.id;

7. STORED PROCEDURE

A stored procedure is a prepared SQL code that you can save and reuse.

CREATE PROCEDURE procedure_name
AS
BEGIN
    SQL statements
END;

Example:

CREATE PROCEDURE GetEmployeeDetails
AS
BEGIN
    SELECT * FROM employees;
END;

8. TRIGGER

A trigger is a set of SQL statements that automatically "fires" when a specified event occurs on a specified table or view.

CREATE TRIGGER trigger_name
ON table_name
AFTER INSERT, UPDATE, DELETE
AS
BEGIN
    SQL statements
END;

Example:

CREATE TRIGGER AuditChanges
ON employees
AFTER INSERT, UPDATE, DELETE
AS
BEGIN
    INSERT INTO audit_log (action_type, action_description, action_date)
    VALUES ('CHANGE', 'Employee record modified', GETDATE());
END;

9. RELATIONSHIP

9.1 One-to-One

A one-to-one relationship between two tables means that each row in the first table can be linked to one and only one row in the second table and vice versa. This is usually implemented using a unique foreign key.

CREATE TABLE Person (
    PersonID INT PRIMARY KEY,
    Name NVARCHAR(100)
);

CREATE TABLE PersonDetail (
    PersonID INT PRIMARY KEY,
    Address NVARCHAR(255),
    FOREIGN KEY (PersonID) REFERENCES Person(PersonID)
);

9.2 One-to-Many

A one-to-many relationship means that a row in one table can be related to multiple rows in another table. This is achieved by placing a foreign key in the related table.

CREATE TABLE Department (
    DepartmentID INT PRIMARY KEY,
    DepartmentName NVARCHAR(100)
);

CREATE TABLE Employee (
    EmployeeID INT PRIMARY KEY,
    Name NVARCHAR(100),
    DepartmentID INT,
    FOREIGN KEY (DepartmentID) REFERENCES Department(DepartmentID)
);

9.3 Many-to-Many

A many-to-many relationship means that a row in one table can be related to multiple rows in another table and vice versa. This is implemented using a junction table that contains foreign keys to both related tables.

CREATE TABLE Student (
    StudentID INT PRIMARY KEY,
    Name NVARCHAR(100)
);

CREATE TABLE Course (
    CourseID INT PRIMARY KEY,
    CourseName NVARCHAR(100)
);

CREATE TABLE StudentCourse (
    StudentID INT,
    CourseID INT,
    PRIMARY KEY (StudentID, CourseID),
    FOREIGN KEY (StudentID) REFERENCES Student(StudentID),
    FOREIGN KEY (CourseID) REFERENCES Course(CourseID)
);
